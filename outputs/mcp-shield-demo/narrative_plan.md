# MCP Shield Demo Deck Narrative Plan

Audience: product stakeholders, engineering reviewers, QA, and business users who need to understand the MCP Shield dashboard and the new C-Admin scope behavior.

Objective: create a concise demo presentation that explains what the project is, how the roles differ, and what to demonstrate live.

Narrative arc:

1. Introduce MCP Shield as a role-aware security analytics dashboard.
2. Explain the three account experiences: Admin, Client, and C-Admin.
3. Focus on the C-Admin problem: delegated access without global admin rights.
4. Show the implementation model: assigned clients, assigned services, scoped onboarding, and scoped reports.
5. Close with a live demo path and QA checklist.

Slide list:

1. Title and demo framing.
2. Product at a glance.
3. Role model.
4. C-Admin access model.
5. Service onboarding flow.
6. Reporting ownership rules.
7. Live demo script.
8. QA validation checklist.
9. Roadmap to production backend.

Visual system: clean product-demo style with teal accent, graphite text, light panels, editable cards, metric blocks, and simple role/flow diagrams. All important slide text is editable.

Source plan: use `PROJECT_DOCUMENTATION.md`, local source files, and current repository behavior. No external sources are required.

Editability plan: build visible copy, cards, tables, and diagrams as editable PowerPoint text and shapes.
